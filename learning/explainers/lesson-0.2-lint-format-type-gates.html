Lint, Format, Type Gates

0.1 gave you an installable, reproducible package. 0.2 makes it self-enforcing: a linter/formatter and a type checker wired into both your editor and pre-commit, so drift gets caught at save-time and commit-time — before it ever reaches a review or a training run.

Why this lesson now? Every lesson from here on adds real code — sim mechanics, then MAPPO internals with tensor shapes that are easy to get subtly wrong. A type checker catches a transposed obs/action shape before you burn a GPU-hour discovering it at runtime. A lint/format gate keeps a solo, lesson-by-lesson codebase from accreting inconsistency across 40+ future sessions. Cheap insurance, bought once.

1 · ruff — one tool, two jobs

ruff is a single Rust binary that does both linting (ruff check) and formatting (ruff format) — replacing the old flake8 + isort + black stack from your src-layout lesson's intro. It's fast enough to run on every save with no perceptible lag.

ruff check
Static analysis: unused imports, undefined names, import order, hundreds of opt-in rule sets (pyflakes, pycodestyle, pyupgrade, bugbear, …).
ruff format
Opinionated code formatter, black-compatible output. No config bikeshedding — that's the point.
# configure once, in pyproject.toml
[tool.ruff]
line-length = 100
target-version = "py312"

[tool.ruff.lint]
select = ["E", "F", "I", "UP", "B"]   # pyflakes, pycodestyle, isort, pyupgrade, bugbear

# run it
uv run ruff check .        # lint
uv run ruff format .       # format
uv run ruff check --fix .  # auto-fix what's safely fixable

2 · ty — type checking, with a caveat

ty is Astral's new type checker (same team as uv/ruff), written in Rust for speed, but it is explicitly beta as of 2025 — expect false positives/negatives and a smaller rule surface than mature checkers. That's why the locked design treats it as non-blocking in CI for now, with mypy as the documented fallback if ty's gaps get in the way on real code (e.g. once PyTorch tensor typing gets involved in Phase 4).

uv run ty check .          # or: uvx ty check .
Practical stance: run ty locally and treat its output as signal, not law, while it's beta. Don't spend time fighting it to satisfy an edge case — note the gap and move on. This is a deliberate, documented trade-off, not carelessness.

3 · Pre-commit: the gate that can't be skipped by forgetting

A lint rule you have to remember to run is a lint rule you'll eventually forget to run. pre-commit installs a git hook that runs your checks automatically on every git commit, blocking the commit if they fail.

# .pre-commit-config.yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.x.x          # pin an actual released tag
    hooks:
      - id: ruff-check
        args: [--fix]
      - id: ruff-format
uv add --dev pre-commit
uv run pre-commit install       # writes .git/hooks/pre-commit
uv run pre-commit run --all-files   # dry-run against the whole repo once

Note: ty is intentionally not in the pre-commit gate yet (per the beta caveat above) — keep it as a manual/CI-informational check for now, wired into your editor for immediate feedback instead.

4 · Editor integration

The fastest feedback loop is the one that never leaves your editor. If you're on VS Code: install the official Ruff extension (lint + format-on-save) and, if available for your editor, Astral's ty language-server extension for inline type diagnostics. Point your editor's Python interpreter at .venv (uv creates it) so import resolution matches what uv run actually sees.

6 · Your exercise you code this

Wire the gates into the repo you already have. End-state:

  • [tool.ruff] + [tool.ruff.lint] sections added to pyproject.toml with an explicit select list (at least pyflakes/pycodestyle/isort).
  • uv run ruff check . and uv run ruff format --check . both pass clean on the current repo.
  • uv run ty check . runs (output reviewed — doesn't need to be zero-issue given the beta caveat, but you should be able to explain any remaining findings).
  • pre-commit added as a dev dependency, a .pre-commit-config.yaml with at least the ruff-check + ruff-format hooks, and pre-commit install run so the git hook exists.
  • Prove the gate works: make a trivial lint violation (e.g. an unused import), attempt git commit, and confirm it's blocked — then fix it and commit successfully.
  • Editor wired for at least format-on-save (screenshot/description is fine if I can't see your editor directly).
