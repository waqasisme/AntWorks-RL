# Hydra configures the root logger — `setup_logging` must force its handlers

During 0.5 review, `logging.renderer=json` appeared to "work" but emitted Hydra's default format wrapping a Python-`dict` repr, and `logging=debug` never showed debug lines. Root cause: `@hydra.main` installs its own handler on the root logger *before* `main()`'s body runs, so `logging.basicConfig(...)` — documented to no-op when the root already has handlers — silently did nothing; neither the custom `ProcessorFormatter` nor the configured level took effect. Fix: `logging.basicConfig(..., force=True)` (or reset the root logger's handlers/level directly) so `setup_logging` reliably owns the root sink regardless of Hydra. This is a standing implication of the locked Hydra choice: any component that configures logging must assume Hydra got there first.
