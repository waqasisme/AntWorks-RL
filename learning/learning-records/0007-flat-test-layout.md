# Tests are flat modules with unique basenames — no stdlib-shadowing packages

During 0.5, a `tests/logging/` subpackage broke collection two ways under pytest's default `prepend` import mode: with no `tests/__init__.py`, the walk-up made the module's top-level name `logging`, shadowing the stdlib (`ModuleNotFoundError: No module named 'logging.test_logging'`); and a duplicate `test_logging.py` at two paths triggered an "import file mismatch". Convention adopted: keep tests as flat files directly under `tests/` (`test_config.py`, `test_logging.py`, `test_rng.py`) with unique basenames and no test-package `__init__.py` — matching the existing `test_rng.py`. Also relevant: a test using Hydra's `initialize(config_path=...)` is location-sensitive (relative to the calling module, per [[0004-hydra-relative-config-path]]), so moving it changes the path — another reason to keep the layout flat and stable.
